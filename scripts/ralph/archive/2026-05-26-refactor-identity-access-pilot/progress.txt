## Codebase Patterns
- Bounded contexts live at backend/contexts/<name>/ with a narrow public API per module and a thin api.py router; mirrored at frontend/src/contexts/<name>/{api.ts, ui.tsx}.
- Shared infra modules at backend root: db.py (Supabase/asyncpg factories), auth.py (JWT + permissions), outbox.py (single emit function). Never inline these — import them.
- Migrations go in migrations/NNN_<name>.sql; apply with a paired backend/apply_<name>_migration.py following the existing pattern (see migrations 001-017).
- RLS is the access boundary — every new table mirrors the participations-based pattern. Tests must verify that a user outside participations cannot CRUD.
- Tests are colocated next to the module (test_<module>.py in the same directory). Pure-function modules get exhaustive table-driven pytest with no DB.
- Frontend test files colocate as ui.test.tsx / api.test.ts with vitest; mock '@/lib/supabase' when the module under test transitively imports it.
- "Robust CRUD" acceptance bar: form validates required fields, surfaces server errors inline, disables submit in-flight, renders empty/loading states, accessible labels, and shows confirm-before-destructive dialogs.
- Hard-delete vs retire decision lives in the asset_lifecycle pure module; never duplicate the count-then-decide logic.
- Tree validation (parent-kind / child-kind) lives in tree_validation pure module; reused across Space + System.
