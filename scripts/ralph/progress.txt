## Codebase Patterns
- Backend asset column additions: update _ASSET_COLUMNS string, add param to create_asset/update_asset signatures, add to INSERT columns+VALUES (bump $N), add to UPDATE SET (bump $N), update _asset_row helper in tests
- apply_*_migration.py pattern: psycopg2 + load_dotenv('../.env.local'), open migration file, execute, verify column exists
- typecheck for backend: not applicable (Python), run pytest instead
- frontend-kit primitives: import from `@/lib/frontend-kit`; all CSS classes are fk-* (never bp-*); styles auto-loaded via layout.tsx
- frontend-kit pattern: components use className prop pass-through + variant/size/color unions; no `any` types; `'use client'` at top
- WBar/WTabs/WKVGrid do NOT accept `style` prop — use `className` with JSX `<style jsx>` CSS instead
- Tab state: persist in URL with `router.replace()` + `?tab=TabName`; read with `useSearchParams().get('tab')`
- Detail page data loading: fetch instances with `{ asset_id: assetId }` filter; call `listTemplates()` once and build `Map<id, Template>` for name resolution
- PhaseTracker maps phase string → current step via PHASE_TO_CURRENT_STEP dict; connector lines use `idx <= currentIdx` for done vs upcoming
