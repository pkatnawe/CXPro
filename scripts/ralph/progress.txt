## Codebase Patterns
- useUrlFilters hook: avoid useCallback (React internals won't work in direct vitest calls); use plain nested functions that close over router/searchParams
- Next.js router mock in vitest: `vi.mock('next/navigation', () => ({ useRouter: () => ({ replace: mockFn }), useSearchParams: () => new URLSearchParams(str) }))`
- Backend filter params: add as optional keyword arg with `str | None = None`, append to conditions list with parameterized `$N` placeholder, increment idx
- Tests use `_mock_conn()` returning MagicMock with AsyncMock fetch/fetchrow; no live DB needed
- All backend instance tests are in `backend/contexts/commissioning_execution/test_instances.py`
- Frontend pure modules (no React/Next/fetch) live in `frontend/src/contexts/navigation/`
- Vitest tests: table-driven with `describe/it/expect`, no mocks needed for pure functions
- typecheck command is `npm run type-check` (not `typecheck`)
- React context split pattern: outer component provides `<Provider>`, inner component uses `useContext` — can't do both in one component
- BreadcrumbLabel context: `useBreadcrumbLabel()` from `@/contexts/navigation/breadcrumbLabel` — detail pages call `setEntityLabel` in a useEffect to inject crumb label into topbar
