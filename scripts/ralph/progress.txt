## Codebase Patterns
- Database migrations are applied using Python scripts in backend/ directory (e.g., apply_migration.py)
- The function signature of Postgres functions should remain unchanged when updating their body
- Test database functions by calling them directly with SQL via psycopg2
- TypeScript type checking is done with `npm run type-check` in frontend directory
- Integration tests use asyncpg for async database operations
- The handle_new_user trigger fires AFTER INSERT on auth.users and automatically redeems pending invitations
- FastAPI endpoints: Use `Depends(security)` with HTTPBearer for JWT auth, pydantic BaseModel for request bodies
- Supabase client setup: Create with `create_client(url, service_key)`, methods are synchronous (not async)
- Backend services: Create service classes that accept supabase client and db connection in __init__
- Test with unique identifiers (e.g., 'test-org-inv' instead of 'test-org') to avoid conflicts
