## Codebase Patterns
- commissioning_execution frontend context lives at frontend/src/contexts/commissioning_execution/ (api.ts, ui.tsx, ui.test.tsx)
- AssetTypeLinkPicker determines linked asset types by querying listTemplates(projectId, {asset_type_id}) for each asset type — no dedicated "list links" endpoint
- Migrations live in /migrations/ (root level), numbered with prefix e.g. 018_asset_registry.sql
- Apply scripts live in backend/ as apply_<name>_migration.py, load dotenv from '../.env.local', use psycopg2
- RLS pattern: participations-based — SELECT/INSERT/UPDATE/DELETE check EXISTS (SELECT 1 FROM participations WHERE project_id = <table>.project_id AND user_id = auth.uid())
- For child tables without project_id (points, asset_system_memberships), RLS joins via parent table to participations
- Partial unique index for soft-delete: CREATE UNIQUE INDEX ... WHERE status != 'retired'
- Existing failing tests (pre-existing): test_invitation_service.py, test_invitation_service_simple.py, test_redeem_pending_invitations.py, tests/test_invites_endpoint.py (need DB integration) — ignore these
- CRUD modules use asyncpg.Connection; tests mock it with MagicMock + AsyncMock (no live DB needed)
- FastAPI routers: prefix on router (e.g., /projects/{project_id}), register in main.py via app.include_router()
- UUID params passed as uuid.UUID() to asyncpg; str() or hasattr(v, "hex") used to serialize back to string in API layer
- update_space uses sentinel `...` for optional params to distinguish "not provided" from None (for nullable fields)
- Frontend API modules call the backend via fetch() with Bearer token from supabase.auth.getSession(); base URL from NEXT_PUBLIC_API_URL || 'http://localhost:8000'
- Frontend tests use vitest + jsdom; no @testing-library/react — test logic/API functions directly with vi.stubGlobal('fetch', ...) mocks
- vi.mock() inside beforeEach() causes hoisting issues; always place vi.mock() at top level of test file
- asyncpg conn.transaction() is an async context manager; mock it with @asynccontextmanager async def _transaction(): yield (assigned to conn.transaction)
- Frontend pages use 'use client', useParams/useRouter, and inline <style jsx> CSS (no Tailwind classes in page components)
- SpaceKind union and ALLOWED_PARENTS mirror backend tree_validation.py exactly — keep in sync
